# Инструкция по драйверу PostgreSQL (PostgresDriver)

Драйвер — это модуль `postgres_driver.py`, который даёт единый способ подключения к PostgreSQL и выполнения операций **Create, Read, Update, Delete (CRUD)** из любого проекта.

---

## 1. Установка и подключение в проекте

### Зависимости

В проекте должны быть установлены:

- `psycopg2-binary` — работа с PostgreSQL
- `python-dotenv` — загрузка настроек из файла `.env`

Установка:

```bash
pip install psycopg2-binary python-dotenv
```

### Подключение драйвера в коде

Скопируйте файл `postgres_driver.py` в папку вашего проекта (или добавьте путь к нему в `sys.path`), затем:

```python
from postgres_driver import PostgresDriver

db = PostgresDriver()
db.connect()
# ... работа с БД ...
db.close()
```

Либо используйте контекстный менеджер (соединение закроется автоматически):

```python
from postgres_driver import PostgresDriver

with PostgresDriver() as db:
    # ... работа с БД ...
    pass  # при выходе из with соединение закроется
```

### Файл `.env`

Параметры подключения берутся из переменных окружения. Удобнее всего задать их в файле `.env` в корне проекта:

```env
POSTGRES_HOST=localhost
POSTGRES_PORT=5432
POSTGRES_DB=postgres
POSTGRES_USER=postgres
POSTGRES_PASSWORD=ваш_пароль
```

По умолчанию драйвер ищет файл `.env` в той же папке, где лежит `postgres_driver.py`. Чтобы указать другой путь:

```python
from pathlib import Path
from postgres_driver import PostgresDriver

db = PostgresDriver(env_path=Path("/путь/к/проекту/.env"))
db.connect()
```

Или при вызове `connect()`:

```python
db.connect(env_path=Path("/путь/к/.env"))
```

## 2. Методы драйвера (CRUD)

Во всех методах имя таблицы передаётся строкой. Условия и данные — словарями `{имя_колонки: значение}`. Параметры подставляются через `%s`, поэтому риск SQL-инъекций минимален при использовании только этих методов.

---

### Create (создание записи) — `create(table, data)`

Вставляет одну строку в таблицу.

- **table** — имя таблицы (строка).
- **data** — словарь: ключи — имена колонок, значения — то, что вставляем.

Возвращает вставленную строку как словарь (через `RETURNING *`), либо `None`, если сервер не вернул строку.

**Пример:**

```python
row = db.create("users", {"name": "Анна", "email": "anna@example.com"})
print(row)  # например: {'id': 1, 'name': 'Анна', 'email': 'anna@example.com'}
```

---

### Read (чтение) — `read(table, columns, where, order_by, limit)`

Выполняет `SELECT` и возвращает список словарей (каждая строка — один словарь).

- **table** — имя таблицы.
- **columns** — строка с именами колонок или `"*"` (по умолчанию `"*"`).
- **where** — необязательный словарь: условие `WHERE` в виде `{колонка: значение}` (все условия объединяются через `AND`).
- **order_by** — необязательная строка, например `"id DESC"`.
- **limit** — необязательное целое число (максимум строк).

**Примеры:**

```python
# Все записи из таблицы users
all_users = db.read("users")

# Только колонки name и email, первые 10 записей
users = db.read("users", columns="name, email", limit=10)

# Один пользователь по id
user = db.read("users", where={"id": 1})
# user — список из одного словаря, например [{'id': 1, 'name': 'Анна', ...}]

# Сортировка и условие
users = db.read("users", where={"email": "anna@example.com"}, order_by="id DESC")
```

---

### Update (обновление) — `update(table, data, where)`

Обновляет строки в таблице по условию.

- **table** — имя таблицы.
- **data** — словарь с колонками и новыми значениями.
- **where** — обязательный словарь: по каким строкам обновлять (иначе можно случайно обновить всю таблицу).

Возвращает количество обновлённых строк.

**Пример:**

```python
count = db.update("users", {"name": "Анна Петрова"}, where={"id": 1})
print(f"Обновлено строк: {count}")
```

---

### Delete (удаление) — `delete(table, where)`

Удаляет строки по условию.

- **table** — имя таблицы.
- **where** — обязательный словарь (условие). Пустое условие не допускается, чтобы избежать случайного удаления всех строк.

Возвращает количество удалённых строк.

**Пример:**

```python
count = db.delete("users", where={"id": 1})
print(f"Удалено строк: {count}")
```

---

### Произвольный SQL — `execute(sql, params)`

Если нужен свой запрос (сложный `SELECT`, вызов функции и т.п.):

- **sql** — строка с SQL; места для параметров — `%s`.
- **params** — необязательный кортеж параметров по порядку.

Для `SELECT` возвращает список словарей (результат). Для `INSERT`/`UPDATE`/`DELETE` возвращает пустой список.

**Пример:**

```python
result = db.execute("SELECT * FROM users WHERE id = %s", (1,))
custom = db.execute("SELECT name, count(*) as cnt FROM users GROUP BY name")
```

## 3. Краткая шпаргалка

| Действие        | Метод                                      |
|-----------------|--------------------------------------------|
| Подключиться    | `db.connect()` или `with PostgresDriver() as db:` |
| Отключиться     | `db.close()`                               |
| Вставить строку| `db.create("таблица", {"колонка": значение})` |
| Выбрать строки  | `db.read("таблица", where={...}, limit=10)` |
| Обновить        | `db.update("таблица", {"колонка": новое}, where={...})` |
| Удалить         | `db.delete("таблица", where={...})`        |
| Свой SQL        | `db.execute("SELECT ... WHERE id = %s", (1,))` |

---

## 4. Важно

- В **update** и **delete** всегда передавайте **where** — пустое условие в этих методах намеренно запрещено.
- Файл `.env` с паролем не должен попадать в репозиторий (добавьте `.env` в `.gitignore`). Для примера настроек используйте `.env.example` без секретов.
- Драйвер при первом вызове CRUD сам вызывает `connect()`, если соединение ещё не установлено (при использовании того же пути к `.env`).

Если что-то пойдёт не так, проверьте: запущен ли PostgreSQL, верны ли данные в `.env`, существует ли таблица и корректны ли имена колонок.
